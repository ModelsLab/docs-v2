Generate videos from text descriptions using state-of-the-art video generation models like CogVideoX. Perfect for creating short-form content, animations, and visual storytelling.

## Request

Make a `POST` request to the endpoint below with the required parameters.

```bash
POST https://modelslab.com/api/v6/video/text2video
```

## Parameters

<ParamField body="key" type="string" required>
  Your API key for authentication.
</ParamField>

<ParamField body="model_id" type="string" required>
  Video generation model to use. Options: `cogvideox`, `animatediff`, `svd`.
</ParamField>

<ParamField body="prompt" type="string" required>
  Text description of the video you want to generate. Be descriptive about motion and action.
</ParamField>

<ParamField body="negative_prompt" type="string">
  Things you don't want in the video. Example: "low quality, blurry, static"
</ParamField>

<ParamField body="width" type="integer" default="512">
  Video width in pixels. Must be divisible by 8.
</ParamField>

<ParamField body="height" type="integer" default="512">
  Video height in pixels. Must be divisible by 8.
</ParamField>

<ParamField body="num_frames" type="integer" default="25">
  Number of frames to generate. More frames = longer video.
</ParamField>

<ParamField body="num_inference_steps" type="integer" default="20">
  Number of denoising steps. Higher = better quality, slower generation.
</ParamField>

<ParamField body="guidance_scale" type="float" default="7">
  How closely to follow the prompt. Range: 1-20.
</ParamField>

<ParamField body="output_type" type="string" default="mp4">
  Output format. Options: `mp4`, `gif`.
</ParamField>

<ParamField body="fps" type="integer" default="8">
  Frames per second in the output video.
</ParamField>

<ParamField body="upscale_height" type="integer">
  Height to upscale the video to after generation.
</ParamField>

<ParamField body="upscale_width" type="integer">
  Width to upscale the video to after generation.
</ParamField>

<ParamField body="upscale_strength" type="float" default="0.6">
  Strength of upscaling. Range: 0-1.
</ParamField>

<ParamField body="webhook" type="string">
  URL to receive results when generation completes. See [Webhooks](/webhooks).
</ParamField>

<ParamField body="track_id" type="string">
  Your custom identifier to track this request.
</ParamField>

```python Python
import requests
import time

# Make the request
response = requests.post(
    "https://modelslab.com/api/v6/video/text2video",
    json={
        "key": "your_api_key",
        "model_id": "cogvideox",
        "prompt": "A majestic space station orbiting Earth, cinematic, 4K",
        "negative_prompt": "low quality, blurry",
        "width": 512,
        "height": 512,
        "num_frames": 25,
        "num_inference_steps": 20,
        "guidance_scale": 7,
        "output_type": "mp4"
    }
)

data = response.json()

# Video generation is async - poll for results
if data.get("status") == "processing":
    request_id = data["id"]
    while True:
        fetch_response = requests.post(
            f"https://modelslab.com/api/v6/video/fetch/{request_id}",
            json={"key": "your_api_key"}
        )
        result = fetch_response.json()
        if result.get("status") == "success":
            print(f"Video URL: {result['output'][0]}")
            break
        elif result.get("status") == "failed":
            print(f"Generation failed: {result.get('message')}")
            break
        time.sleep(5)  # Wait 5 seconds before polling again
else:
    print(data)
```

## Response

### Success Response

When the video is generated immediately:

```json
{
  "status": "success",
  "generationTime": 45.2,
  "id": 12345678,
  "output": [
    "https://pub-3626123a908346a7a8be8d9295f44e26.r2.dev/generations/video123.mp4"
  ],
  "meta": {
    "prompt": "A majestic space station orbiting Earth...",
    "model_id": "cogvideox",
    "width": 512,
    "height": 512,
    "num_frames": 25
  }
}
```

### Processing Response

Video generation typically takes longer, so you'll usually receive a processing status:

```json
{
  "status": "processing",
  "id": 12345678,
  "eta": 60,
  "message": "Your video is being generated"
}
```

<Note>
Video generation can take 30-120 seconds depending on length and quality settings. Use the [Fetch endpoint](/video-api/fetch-video) to poll for results, or use [webhooks](/webhooks) for async notification.
</Note>

### Error Response

```json
{
  "status": "error",
  "message": "prompt is required"
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | "success", "processing", or "error" |
| `id` | integer | Unique request identifier for fetching results |
| `output` | array | URLs of generated videos (when complete) |
| `eta` | integer | Estimated time to completion in seconds |
| `generationTime` | float | Time taken to generate (seconds) |

## Async Pattern

Since video generation takes time, use this pattern:

```python
import requests
import time

def generate_video(prompt, api_key):
    # 1. Submit the request
    response = requests.post(
        "https://modelslab.com/api/v6/video/text2video",
        json={
            "key": api_key,
            "model_id": "cogvideox",
            "prompt": prompt,
            "num_frames": 25
        }
    )
    data = response.json()

    if data["status"] == "error":
        raise Exception(data["message"])

    request_id = data["id"]

    # 2. Poll for results
    while True:
        fetch = requests.post(
            f"https://modelslab.com/api/v6/video/fetch/{request_id}",
            json={"key": api_key}
        )
        result = fetch.json()

        if result["status"] == "success":
            return result["output"][0]
        elif result["status"] == "failed":
            raise Exception(result.get("message", "Generation failed"))

        # Still processing, wait and retry
        time.sleep(5)

# Usage
video_url = generate_video("A sunset over the ocean", "your_api_key")
print(f"Video ready: {video_url}")
```

## Tips for Better Videos

<AccordionGroup>
  <Accordion title="Describe Motion">
    Unlike images, videos need motion descriptions:
    - ❌ "A cat"
    - ✅ "A cat walking across a sunny room, tail swaying"
  </Accordion>

  <Accordion title="Keep It Simple">
    Video models work best with clear, focused prompts. Avoid overly complex scenes.
  </Accordion>

  <Accordion title="Use Upscaling">
    Generate at 512x512 then use upscale parameters for higher resolution output.
  </Accordion>

  <Accordion title="Choose Output Format">
    - **MP4**: Best for most uses, smaller file size
    - **GIF**: Good for short loops, works everywhere
  </Accordion>
</AccordionGroup>
